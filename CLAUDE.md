# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

"Platforma za roditelje" — a static informational site (in Serbian) aimed at parents, with content on school grading, counseling, FAQs, and the team behind the platform.

## Build / serve

The site is built with [Eleventy (11ty)](https://www.11ty.dev/). There is no `package.json`, `.eleventy.js`, or lockfile — Eleventy is run ad-hoc via `npx`:

```sh
npx @11ty/eleventy --serve   # dev server with live reload
npx @11ty/eleventy            # one-shot build into _site/
```

`_site/` is the build output (committed snapshots present, but regenerated by the build). `README.md` is excluded from the build via `.eleventyignore`.

## Structure

- Each top-level `.md` file at the repo root is a page (e.g. `index.md`, `ocenjivanje-u-skoli.md`, `savetovanje.md`, `pitanja-i-odgovori.md`, `o-nama.md`). Eleventy maps `foo.md` to `/foo/` in the output.
- Internal links use slugged URLs without `.md` (e.g. `/savetovanje`, `/pitanja-i-odgovori`). When adding a new page, add a matching link in `index.md`.
- Content within pages may reference external documents like `Tekst 2.docx`, `Tekst 4.docx`, `Tekst 5.docx` — these are placeholders for materials not yet in the repo.
- `inspiration/` holds a saved reference website (`website.html` + `website_files/`) used for design/content inspiration only — it is not part of the build. The real brand palette/typography is in the inline `<style id="global-styles-inline-css">` block of `website.html` (Elementor kit-9), NOT in the CSS files under `website_files/` (those are Hello Elementor + plugin defaults).
- `design-system/` holds the extracted foundation: `tokens.css` (CSS custom properties — colors, typography, spacing, radii, shadows) and `index.html` (single-page reference doc that consumes the tokens). Open `design-system/index.html` directly in a browser to review.

## Design tokens

- All design values live in `design-system/tokens.css` as `:root` custom properties. Project naming is `--color-{role}`, `--font-size-{step}`, `--space-{n}`, `--radius-{name}`, `--shadow-{level}`.
- Tokens are aligned to Bootstrap's `--bs-*` convention: project tokens are mapped into `--bs-primary`, `--bs-body-color`, `--bs-body-bg`, `--bs-border-radius`, etc. The intention is to layer `bootstrap.css` on top later; until then, use the project tokens directly. When adding new tokens, mirror them into the matching `--bs-*` variable if Bootstrap has one.
- Brand palette is sourced verbatim from the Elementor kit: primary `#244F95` (deep blue), accent `#DDB33D` (warm gold), plus teal/taupe/red/navy accents and a 0–900 neutral scale.
- Typography is Montserrat only (weights 400/500/600/700/800), imported from Google Fonts at the top of `tokens.css`. A commented Work Sans alternative is included for body copy if we ever want two families.

All user-facing content is in Serbian (Latin script). Preserve diacritics (š, č, ć, ž, đ) when editing. The `ocenjivanje-u-skoli.md` page is the only substantive content so far; other pages are stubs.
